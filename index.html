Version 2.x of the Carneades argumentation system provides web-based, collaborative software tools for:

  • summarizing the arguments of a debate in an argument graph
  • visualizing, browsing and navigating argument graphs
  • critically evaluating arguments
  • forming opinions, participating in polls and ranking stakeholders by the degree to which they share your views
  • obtaining clear explanations, using argument graphs, of the different effects of alternative policies in particular cases

Carneades is Open Source software licensed under the European Union Public License (EUPL v1.1), which is compatible with the GNU GPL (version 2), and the Eclipse Public License (verision 1.0), among others.

Getting Started

Prerequisites:

  • Version 7 or better of a Java Runtime Environment (JRE 7).
  • A recent version of a standards-compliant Web Browser (e.g. Firefox, Chrome, Safari).

To download, install and run the Carneades Web Application

  1. Download the carneades-webapp.zip file.
  2. Unzip the carneades-webapp.zip file.
  3. Change to the carneades-webapp directory.
  4. Double click on the carneades-webapp.jar file.
  5. Alteratively, execute the following command line:
    $ java -jar carneades-webapp.jar
  6. The server will be start and open the home page of the Carneades client in your web browser.
  7. If the home page fails to open, enter this URL into the address bar of your web browser:
    http://localhost:8080/policymodellingtool/home.html
  8. To shutdown the server you may need to end the Java task using your task manager, depending on your operating system. If you started the server from the command line, you can kill the process with Control-C on Unix machines, including Macs. If you started the server on a Mac by double clicking on the jar file, you can shutdown the server by quitting the "routes_selfexe" application in the usual way, with Command-Q.

System Architecture

Carneades 2.x is a Web application with a three-tiered architecture consisting of a relational database backend layer, an application logic layer and a graphical user interface layer. (The previous version of Carneades, a desktop application called the Carneades Editor, is still available in the downloads section.)

The server-side of Carneades is implemented in the Clojure dialect of Lisp, which is compiled to bytecode for the Java Virtual Machine (JVM). A RESTful web service provides Internet access to the application logic. Data is interchanged between the server and clients using the JavaScript Object Notation (JSON).

The client-side is implemented as a Rich Internet Application (RIA) using W3C standards, in particular Asynchronous JavaScript and XML (Ajax). The graphical user interface of the web client is implemented using the jQuery JavaScript library.

The web-based version of Carneades was developed in the European IMPACT governance and policy modeling project. The following modifications and extensions of Carneades were realized in the IMPACT project:

  • Carneades was first ported from Scheme to Clojure, another Lisp dialect, to enable it to run on the Java Virtual Machine and make use of the extensive set of available Java libraries, for example for interacting with databases via the Java Database Connectivity library (JDBC) and for implementing web services and applications.
  • Next we have refactored Carneades to more cleanly separate its modules for argument construction and argument evaluation. It is now easier to implement and "plug in" modules for other computational models of structured argument.
  • To validate this architecture we developed an evaluator for argument graphs via a mapping to Dung abstract argumentation frameworks [Dung, 1995], inspired by ASPIC+ [Prakken, 2010a], but extended to support the feaures of Carnades for handling proof burdens and standards [Gordon, Prakken, and Walton, 2007a]. This new evaluator can now handle cyclic argument graphs. Currently grounded semantics has been implemented, but the system has been designed to be easily extensible to support any Dung semantics.
  • We have designed and implemented a Domain Specific Language (DSL) in Clojure for visualising directed graphs, similar to the "dot" language used by the Graphviz system [Ellson, 2001], called LACIJ, for the purpose of generating visualisations of argument graphs for the Web, using the Structured Vector Graphics (SVG) web standard for 2D vector graphics. This language is used to visualize argument graphs in the policy modeling tool.
  • A new XML format for interchanging arguments, called the Carneades Argument Format (CAF), has been developed. CAF is based on the argument graph portion of the Legal Knowledge Interchange Format (LKIF) developed in the ESTRELLA project, but has been extended to provide better support for metadata about the sources of arguments and simplified by omitting the parts of LKIF for representing rules. Metadata is represented using the Dublin Core Metadata Element Set. By design, CAF is isomorphic to the abstract computational (algebraic) model of Carneades argument graphs and is intended to be used to represent and serialize argument graphs in a portable fashion. In CAF, every argument and statement node of an argument graph is assigned a globally unique Universal Resource Name (URN), to enable statements and arguments to preserve their identity when argument graphs are transported across system boundaries.
  • Argumentation schemes and policies are represented in a new Domain Specific Language (DSL) for defeasible inference rules, based on Clojure symbolic expression (S-Expressions). The language provides a way to specify templates for translating formulas into positive or negative assertions, as well as questions, in several natural languages, as required to support European policy debates. The language facilitates the isomorphic modelling of legislation, as a hierarchy of sections. Rules ("schemes") can be included at every level of the hierarchy. Metadata, using the Dublin core attributes, can be associated with the rulebase as a whole, as well as each section and scheme. This metadata also enables links to the original legal sources to be included within the model.
  • A relational database backend for storing and managing argument graphs has been developed. The database schema is isomorphic to CAF (and thus the algebraic model presented in the "Argument Graphs" chapter of this report), to enable CAF files to be imported into a database, and exported from a database, with no loss of information.
